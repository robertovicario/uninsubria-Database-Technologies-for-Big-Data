# Redis

## Data Model

Redis is a **_key-value_** store. It stores data as a collection of key-value pairs, where:

- **Key**: A unique identifier for each data item.
- **Value**: The actual data associated with the key. Values could be categorized into **_primitives_** (strings) and **containers** (complex data structures).

## Data Persistence

Redis guarantees this property by using the following components:

- **Redis Instance**: A **_main memory database_** that holds data and serves client requests.
- **RDB Snapshots**: Periodic point-in-time dumps of the dataset to disk in **_RDB (Redis Database)_** format.
- **AOF Log**: An **_append-only file_** that records every write operation, enabling dataset reconstruction after crashes.

## Data Structures

Redis is a **_data structure server_**. At its core, Redis provides a collection of **_native data types_** that help you solve a wide variety of problems, from caching to queuing to event processing.

Redis implements the following data types:

- **_String_**
- **_Hash_**
- **_List_**
- **_Set_**
- **_Sorted set_**
- Vector set
- Stream
- Bitmap
- Bitfield
- Geospatial
- JSON
- Probabilistic data types
- Time series

## Strings

Strings is an **_array_** data structure **_of bytes_** (or words) that stores a sequence of elements, typically characters, using some character encoding:

### Commands (CLI)

The main commands for the strings are:

- `GET`: Get the value of a key.
- `SET`: Set the value of a key.
- `EXISTS`: Check if a key exists.
- `DEL`: Delete a key.

\vspace{0.5cm}

> **Time Complexity:** All the commands above have a time complexity of $O(1)$, where 1 is the number ($N$) of operations performed.

\vspace{0.5cm}

$\boxed{\textbf{GET}}$

```bash
# --- Syntax ---
GET key
```

```bash
# --- Examples ---
redis> GET nonexisting
(nil)
redis> SET mykey "Hello"
"OK"
redis> GET mykey
"Hello"
```

## Hashes

An hash is a data type that represents a mapping between a string **_field_** and a string **_value_**:

\vspace{0.5cm}

![Hashes Data Structure](img/3/5.png)

### Commands (CLI)

The main commands for hashes are:

- `HGET`: Get the value of a field in a hash.
- `HSET`: Set the value of a field in a hash.
- `HEXISTS`: Check if a field exists in a hash.
- `HDEL`: Delete a field from a hash.

\vspace{0.5cm}

> **Time Complexity:** All the commands above have a time complexity of $O(1)$. The following ones have a time complexity of $O(N)$, where $N$ is the number of operations performed, due to multiple (M) returns.

\vspace{0.5cm}

- `HMGET`: Get the value of a field in a hash.
- `HMSET`: Set the value of a field in a hash.
- `HKEYS`: Get all the keys in a hash.
- `HVALS`: Get all the values in a hash.

$\boxed{\textbf{HGET}}$

```bash
# --- Syntax ---
HGET key field
```

```bash
# --- Examples ---
redis> HSET myhash field1 "foo"
(integer) 1
redis> HGET myhash field1
"foo"
redis> HGET myhash field2
(nil)
```

## Lists

Lists holds collections of **_string_** elements sorted according to their **_order of insertion_**:

\vspace{0.5cm}

![Lists Data Structure](img/3/4.png)

### Commands (CLI)

The main commands for the strings are:

- `LLEN`: Get the length of a list.
- `LPUSH`: Add an element to the beginning of a list.
- `RPUSH`: Add an element to the end of a list.
- `LPOP`: Remove and return the first element of a list.
- `RPOP`: Remove and return the last element of a list.

\vspace{0.5cm}

> **Time Complexity:** All the commands above have a time complexity of $O(1)$. The following ones have a time complexity of $O(N)$.

\vspace{0.5cm}

- `LINDEX`: Get the element at the specified index in a list.
- `LSET`: Set the value of an element in a list by its index.
- `LINSERT`: Insert an element before or after another element in a list.
- `LRANGE`: Get a range of elements from a list.

$\boxed{\textbf{LLEN}}$

## Sorted Sets

Sorted sets contain a unique set of **_members ordered by _** floating-point scores_**:

\vspace{0.5cm}

![Sorted Sets Data Structure](img/3/3.png)

### Commands (CLI)

The main commands for the strings are:

- `ZCARD`: Get the number of members in a sorted set. Its **_time complexity_** is $O(1)$.

\vspace{0.5cm}

> **Time Complexity:** All the commands above have a time complexity of $O(1)$. The following ones have a time complexity of $O(\log(N))$, where $N$ is the number of operations performed, due to multiple returns.

\vspace{0.5cm}

- `ZADD`: Add a member to a sorted set, or update its score if it already exists.
- `ZRANK`: Get the rank of a member in a sorted set, with scores ordered from low to high.
- `ZREM`: Remove a member from a sorted set.

\vspace{0.5cm}

> **Time Complexity:** The following ones have a time complexity of $O(\log(N) + M)$.

\vspace{0.5cm}

- `ZRANGE`: Get a range of members in a sorted set, by index.
- `ZRANGEBYSCORE`: Get a range of members in a sorted set, by score.

## Advanced Features

Redis provides a set of advanced features that allow you to build complex applications on top of it. These features include:

### Transactions

Redis supports transactions to execute multiple commands **_atomically_**:

- A transaction is **_started_** with `MULTI`, which queues commands, and **_executed_** with `EXEC`, which runs all queued operations sequentially.
- If needed, a transaction **_can be canceled_** using `DISCARD`.

Redis **_does not support rollback_**: even if one command fails during execution, the others are still executed.

For concurrency control, `WATCH` can be used to implement **_optimistic locking_** by monitoring keys and aborting the transaction if they are modified before execution.

### Secondary Indexes

Secondary indexes in Redis are used to efficiently query data **_based on attributes_** other than the primary key. They are typically implemented using **_sorted sets_**, where the score represents the **_indexed attribute_** and the value is the object identifier.

### Composite Indexes

Composite indexes allow querying based on multiple attributes **_simultaneously_**. They are often implemented by encoding **_multiple fields into a single value_** (e.g., room:price:id) stored in a sorted set. Queries can then exploit **_lexicographical ordering_** or **_score-based filtering_** to efficiently retrieve matching elements.
